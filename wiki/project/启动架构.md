# AiBrain 启动架构

> 最后更新：2026-05-01

## 概述

AiBrain 采用三层启动架构，确保多次双击不产生重复进程：

```
start.bat
  └→ backend/launcher/start.py     ← 启动入口（清理+拉起）
       └→ backend/launcher/process_manager.py  ← 独立后台进程（管理服务生命周期）
            ├── qdrant.exe          ← 向量数据库
            ├── python app.py --flask-only ← Flask HTTP 后端
            └── python app.py --webview-only ← PyWebView 窗口（可选）
```

---

## 各文件职责

### `start.bat`
- **极简入口**，只有 3 行
- 调用 `start.py`，完成后自动关闭窗口
- 不承担任何业务逻辑

```bat
@echo off
cd /d "%~dp0"
venv312\Scripts\python.exe backend\launcher\start.py %*
```

---

### `backend/launcher/start.py`
**启动前置脚本**，负责：

1. **检查依赖** → 调用 `_boot_helper.py deps`，缺依赖自动 pip install
2. **杀旧进程** → 调用 `kill_old.py` 清理旧 Flask/Qdrant/PM
3. **等待端口释放** → 轮询直到端口空闲（最多15秒），超时后强制通过 netstat 反查 PID 再杀一次
4. **启动新 PM** → `subprocess.Popen(CREATE_NEW_CONSOLE | CREATE_NEW_PROCESS_GROUP)`，弹出独立窗口
5. **自身退出** → start.bat 窗口关闭

---

### `backend/launcher/process_manager.py`
**服务生命周期管理器**，负责：

- **单例锁**：通过 `backend/.pm.pid` 文件保证只有一个 PM 运行
- 启动 Qdrant、Flask、WebView（按顺序）
- 监控子进程崩溃，自动重启
- **检测 `.restart_flask` 标志文件**：每 3 秒轮询，发现后触发 Flask 重启
- 接收 Ctrl+C 优雅退出，按顺序关闭所有子进程

---

### `backend/launcher/kill_old.py`
**旧进程清理脚本**，负责：

- **A1**：netstat 扫描端口，杀占用 Flask/Qdrant 端口的进程
- **A2**：wmic 扫描命令行含 `app.py` 的 Python 进程（Flask + WebView）
- **A3**：wmic 扫描项目目录下的 `qdrant.exe`
- **不清理** `process_manager.py` 进程（由 `start.py` 负责）

---

### `backend/launcher/_boot_helper.py`
**启动辅助工具**：

- `deps` 模式：检查关键依赖是否安装（返回 0/1）
- `ports` 模式：根据项目路径 hash 分配固定端口组，写入 `.port_config`

---

## 端口分配

端口由 `_boot_helper.py` 根据**项目根目录路径的 MD5 hash** 自动分配，不同项目端口不冲突。

端口配置存储于项目根目录 `.port_config`，格式：

```
flask_port,qdrant_http,qdrant_grpc,port4,port5
```

例：`18980,18981,18982,18983,18984`

## 热重载（Flask 重启）

### 触发链路
```
前端点击[重启]
  → POST /flask/restart
      → status.py 写 backend/.restart_flask 标志文件
  → PM monitor() 每 3 秒轮询检测到标志文件
      → os.remove(标志文件)
      → restart_flask() 执行重启
```

### restart_flask() 执行步骤
```
restart_flask()
  1. procs.pop('flask') 移除引用（防止 monitor 双重重启）
  2. netstat -ano 找监听 flask_port 的 PID → taskkill /F /T
  3. 强制杀死 procs 中记录的 Flask 进程（兜底）
  4. socket.bind() 轮询等端口释放（最多 10 次 × 0.5s = 5s）
  5. start_flask() 启动新 Flask
```

### 端口检测方式
- **检测端口空闲**：`socket.bind(('127.0.0.1', port))` 能绑定成功 = 空闲
- **检测端口占用**：绑定失败 = 已被监听
- 轮询直到绑定成功，最多 5 秒

## 单实例保证

| 场景 | 处理方式 |
|------|---------|
| 双击 start.bat（首次） | `start.py` 无旧 PM，直接启动 |
| 双击 start.bat（已有 PM） | `start.py` 找到旧 PM PID → 检查是否存活 → 存活则退出，否则接管 |
| PM 单例锁 | `backend/.pm.pid` 存当前 PID，tasklist 检查是否存活 |
| 热重载触发 | 只重启 Flask，不动 Qdrant 和 PM |
| 手动关闭 PM 窗口 | Ctrl+C 触发优雅退出，按序关闭 Flask → WebView → Qdrant |

## 常见问题

**Q: 双击 bat 后看到空窗口？**
A: start.bat 只是跳板，几秒后自动关闭。真正的服务窗口是弹出的第二个 cmd 窗口（AiBrain Process Manager）。

**Q: 热重载后 MCP 断连？**
A: 热重载只重启 Flask，MCP 进程（brain_mcp/wiki_mcp）不受影响。如果 MCP 断连，在 CodeBuddy 里执行 `/mcp` 重连即可。

**Q: 修改 start.py / process_manager.py 不触发热重载？**
A: 正常，`launcher/` 目录被排除在 watchdog 监控之外，需要重启 PM 才能生效。

**Q: PM 重启后前端无响应？**
A: 检查 `backend/logs/restart_trace.log` 追踪重启过程。
