# AiBrain 架构详细分析

## 项目概述
AiBrain 是一个本地化的个人知识管理与记忆增强系统，集成了记忆存储、向量检索、知识库管理和多模态交互功能。

## 技术架构总览

### 核心架构图
```
AiBrain 系统
├── 前端层 (Web UI)
│   ├── PyWebView (桌面应用外壳)
│   └── Web 静态页面 (HTML/JS/CSS)
├── 后端服务层 (Flask)
│   ├── REST API
│   ├── 模块化路由系统
│   └── 进程管理
├── 记忆引擎层
│   ├── mem0 (记忆管理)
│   ├── Qdrant (向量数据库)
│   └── BGE-M3 (本地嵌入模型)
├── 知识库层
│   ├── LightRAG (知识图谱RAG)
│   ├── Wiki 系统
│   └── 文档索引
├── MCP 服务层
│   ├── brain_mcp (记忆服务)
│   ├── wiki_mcp (知识库服务)
│   └── eye_mcp (屏幕截图服务)
└── 基础设施层
    ├── 进程管理
    ├── 端口分配
    └── 配置管理
```

## 详细架构分析

### 1. 前端架构

#### 1.1 桌面应用外壳
- **技术**: PyWebView
- **作用**: 提供桌面应用体验，封装Web前端
- **实现**: `app.py` 中的 PyWebView 集成
- **特点**: 
  - 支持离线运行
  - 系统托盘集成
  - 多窗口管理

#### 1.2 Web前端
- **技术**: 原生 HTML/JavaScript/CSS
- **架构**: 单页面应用 (SPA)
- **模块化**: 按功能分模块加载
  - `overview/` - 总览页面
  - `memory/` - 记忆管理
  - `steam/` - 活动流
  - `wiki/` - 知识库
  - `settings/` - 设置
  - `logs/` - 日志查看

### 2. 后端架构

#### 2.1 Flask 应用服务器
- **入口**: `backend/app.py`
- **架构**: 模块化路由设计
- **核心模块**:
  - `modules/status.py` - 系统状态监控
  - `modules/memory.py` - 记忆API
  - `modules/wiki_mod.py` - Wiki功能
  - `modules/settings_mod.py` - 设置管理
  - `modules/stats.py` - 统计功能
  - `modules/stream.py` - 活动流

#### 2.2 Core 核心模块
- `core/database.py` - SQLite 数据库管理
- `core/logger.py` - 日志系统
- `core/model.py` - 模型加载管理
- `core/settings.py` - 配置管理

### 3. 记忆引擎架构

#### 3.1 mem0 记忆管理系统
- **位置**: `backend/modules/brain/`
- **核心组件**:
  - `memory.py` - 记忆核心逻辑
  - `mem0_adapter.py` - mem0适配器
  - `organizer.py` - 记忆整理
  - `dedup.py` - 去重功能
  - `migrate.py` - 数据迁移
- **自适应搜索策略**:
  - 数据量 < 100: 全召回模式
  - 数据量 100-1000: 轻过滤 (threshold=0.55)
  - 数据量 > 1000: 精确+rerank (threshold=0.60)

#### 3.2 Qdrant 向量数据库
- **配置**: 动态端口分配
- **存储路径**: `qdrant/storage/`
- **集成方式**: 通过 `brain_mcp/config.py` 配置
- **特点**: 支持增量更新，内存+磁盘混合存储

#### 3.3 BGE-M3 嵌入模型
- **模型位置**: `models/bge-m3/`
- **向量维度**: 1024
- **加载策略**: 按需加载，支持 CPU/GPU 切换
- **离线模式**: 支持完全离线运行

### 4. 知识库架构

#### 4.1 LightRAG 系统
- **位置**: `rag/lightrag_wiki/`
- **核心组件**:
  - `rag_engine.py` - RAG引擎
  - `indexer.py` - 文档索引
  - `config.py` - 配置管理
- **知识图谱**:
  - **实体图**: 文档内实体关系
  - **社区图**: 跨文档宏观关联
- **搜索模式**:
  - **naive**: 纯向量匹配 (0.2s)
  - **local**: 实体图增强 (~15s)
  - **global**: 社区图增强 (~15s)
  - **hybrid**: LLM+图+向量 (~30-45s)
  - **mix**: 多策略融合 (~20s)

#### 4.2 Wiki 管理系统
- **位置**: `wiki/` 目录
- **API**: `backend/modules/wiki_mod.py`
- **功能**: 文档管理、搜索、索引重建
- **集成**: 与 LightRAG 深度集成

### 5. MCP 服务架构

#### 5.1 MCP 服务系统
- **架构**: 多服务独立运行
- **通信**: stdio 协议

#### 5.2 brain_mcp 服务
- **位置**: `brain_mcp/`
- **工具**: `store`, `search`
- **特点**: 零配置自适应搜索

#### 5.3 wiki_mcp 服务
- **位置**: `mcp_servers/wiki_mcp/`
- **工具**: `wiki_search`, `wiki_list`, `wiki_index`
- **功能**: 知识库搜索和管理

#### 5.4 eye_mcp 服务
- **位置**: `mcp_servers/eye_mcp/`
- **功能**: 屏幕截图相关

### 6. 进程管理与启动架构

#### 6.1 多进程管理系统
- **主管理器**: `backend/launcher/process_manager.py`
- **进程树**:
  ```
  process_manager.py (主进程)
  ├── qdrant.exe (向量数据库)
  ├── python app.py --flask-only (API服务)
  └── python app.py --webview-only (UI服务)
  ```

#### 6.2 启动流程
1. **环境检查**: 虚拟环境、依赖
2. **端口分配**: 基于项目路径哈希
3. **进程清理**: 杀掉旧进程
4. **服务启动**: Qdrant → Flask → PyWebView
5. **健康检查**: 等待各服务就绪

#### 6.3 启动脚本
- `start.bat` - Windows批处理启动
- `start.py` - Python启动脚本
- `launch.py` - 高级启动器

### 7. 数据存储架构

#### 7.1 存储目录结构
```
AiBrain/
├── models/           # 嵌入模型
├── qdrant/          # 向量数据库
│   ├── storage/     # 向量数据
│   └── config/      # 配置
├── rag/             # RAG数据
│   └── lightrag_data/
├── logs/            # 日志文件
├── snapshots/       # 数据快照
└── .aibrain/       # 用户配置
```

#### 7.2 数据库设计
- **SQLite 数据库**: `backend/stats.db`
- **表结构**:
  - `daily_stats` - 每日统计
  - `stream` - 活动流记录
  - `search_history` - 搜索历史

### 8. 配置管理系统

#### 8.1 配置层级
1. **环境变量**: 端口、路径等运行时配置
2. **项目配置**: `.port_config`, `settings.json`
3. **用户配置**: `~/.aibrain/config/`
4. **模型配置**: `models/` 目录结构

#### 8.2 动态配置
- **端口分配**: 基于项目路径自动分配
- **设备选择**: CPU/GPU 自动检测
- **模型加载**: 按需加载，支持切换

### 9. 日志与监控

#### 9.1 日志系统
- **多日志文件**: app, flask, ui 分离
- **轮转策略**: 按日期和大小
- **前端集成**: 实时日志查看

#### 9.2 监控指标
- **系统状态**: CPU、内存使用
- **服务状态**: 模型、Qdrant 就绪状态
- **业务指标**: 记忆数量、搜索频率

### 10. 安全与隔离

#### 10.1 进程隔离
- **端口隔离**: 每个项目独立端口
- **进程沙箱**: 子进程独立运行
- **资源限制**: 内存、CPU使用监控

#### 10.2 数据安全
- **本地存储**: 所有数据本地保存
- **离线运行**: 不依赖外部服务
- **权限控制**: 文件系统访问控制

## 架构特点总结

### 优势
1. **完全本地化**: 所有组件本地运行，无外部依赖
2. **模块化设计**: 各组件解耦，易于扩展
3. **自适应优化**: 根据数据量自动调整搜索策略
4. **多模态集成**: 记忆、知识库、截图等多功能集成
5. **进程管理**: 完善的进程生命周期管理

### 技术选型考量
1. **Flask + PyWebView**: 轻量级桌面应用方案
2. **Qdrant + BGE-M3**: 高性能本地向量检索
3. **mem0**: 专业记忆管理框架
4. **LightRAG**: 知识图谱增强检索
5. **MCP 协议**: 标准化工具集成

### 扩展性设计
1. **插件化架构**: 可添加新的 MCP 服务
2. **模块化路由**: 易于添加新功能模块
3. **配置驱动**: 通过配置扩展功能
4. **API 先行**: 所有功能都有 API 接口

## 部署与运行

### 部署架构
1. **单机部署**: 所有组件本地运行
2. **多实例**: 支持多个项目实例并行
3. **便携式**: 无需安装，解压即用

### 运行要求
- **操作系统**: Windows 10/11
- **Python**: 3.12+
- **内存**: 8GB+ (推荐 16GB)
- **存储**: 5GB+ 可用空间
- **显卡**: 可选 (CUDA 支持)
