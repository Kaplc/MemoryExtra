# Shader 阴影

> 创建时间：2026-04-29

## 概述

阴影是渲染技术中的重要组成部分。

## 阴影形成原理

### 核心三要素

1. **投射 (Projection)**
   - 需要有一个光源作为投射源
   - 光线从光源出发照射到物体上

2. **采样 (Sampling)**
   - 对阴影进行采样处理
   - 判断物体表面某点是否处于阴影中

3. **Pass 通道**
   - Unity 的 Pass 通道概念
   - 阴影通过多个 Pass 通道处理
   - 常见的 Pass 如：ShadowCaster、ShadowCollector 等

> 志远指出：这里的"半影"不是指Penumbra，是Unity的Pass通道概念

## 阴影类型

### 1. 硬阴影 (Hard Shadow)
- 边缘清晰
- 计算简单

### 2. 软阴影 (Soft Shadow)
- 边缘柔和
- 更真实的视觉效果

## Shadow Mapping 原理（深度图比较）

### 为什么屏幕深度图可以与光源深度图比较？

**通俗原理：**

1. **屏幕深度图 → 重建3D空间**
   - 屏幕深度图的每个像素点都携带深度值
   - 通过这个深度值可以反推该点在3D空间中的位置

2. **光源方向 = 虚拟摄像机**
   - 光源相当于一个摄像机，从光源角度拍摄场景
   - 生成光源方向的深度图（Shadow Map）

3. **射线投射到光源深度图**
   - 屏幕深度图上的任一点，沿着光源方向发射射线
   - 这条射线会映射到光源深度图的2D平面上

4. **深度值比较判断遮挡**
   - 如果在光源深度图的同一点，**存在深度值比屏幕深度还小的点**
   - 说明该点被中间物体遮挡了 → **产生阴影**
   - 如果光源深度图的深度值 >= 屏幕深度 → **没有被遮挡**

> 志远分享：核心就是通过深度值比较来区分是否被遮挡。屏幕深度图重建3D空间，光源方向相当于摄像机，射线映射到光源深度图，如果发现相同位置存在更小的深度值说明被遮挡。

## Unity/UE 中的实现

### 实际开发中的做法

- **一般阴影处理**：直接用 Unity 的 fallback 内置处理就行
- **不自己写基础阴影**：一般不自己写这种基础的阴影实现
- Unity 提供了内置的阴影系统，大多数情况下直接使用即可

> 志远分享：在实际开发中，一般直接用 Unity 内置的阴影处理，不自己写基础阴影代码。
